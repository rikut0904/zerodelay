# コントリビューションガイド

## 開発フロー

1. Issueを作成または既存のIssueを確認
2. `main`ブランチから作業ブランチを作成
3. 変更を実装
4. Pull Requestを作成

## ブランチ命名規則

```
fix/[機能名]   # 新機能
bug/[修正内容]     # バグ修正
docs/[内容]        # ドキュメント
ref/[内容]    # リファクタリング
```

## コミットメッセージ

```
[種別]/[変更内容]

fix/新機能追加
bug/バグ修正
docs/ドキュメント変更
ref/リファクタリング
test/テスト追加・修正
other/ビルド・設定変更
```

## Pull Request

- 関連するIssue番号を記載
- テンプレートに従って記述
- レビュー前にセルフチェック

## コードスタイル

- プロジェクトの既存コードに合わせる
